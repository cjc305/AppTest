### APT-AUDIT-CRIT-001: AppDeepLink whitelist bypass + ProofViewer URL injection

```yaml
id: APT-AUDIT-CRIT-001
severity: crit
status: done
owner: claude
eta_days: 0
blocker_for: [release_to_play_store_after_security_review]
discovered_via: parallel sub-agent audit + manual verification
fixed_in: pending_commit
related: [APT-AUDIT-CRIT-002, APT-AUDIT-CRIT-003]
```

### Evidence
- `app/src/main/AndroidManifest.xml:55-83` — `<data android:scheme="apptest" />` (no host filter)
- `app/src/main/kotlin/com/apptest/app/nav/AppNavHost.kt:112-170` — no `navDeepLink { uriPattern = ... }` block
- `core/navigation/.../AppDeepLink.kt:36-72` — `parse()` only called from `inboxGraph.onItemDeepLink` (line 151)
- `app/src/main/kotlin/com/apptest/app/proof/ProofViewerScreen.kt:38` — `val url = "https://apptest.dev/v/$proofId.png"` (no validation)

### Fix
```kotlin
// FILE: core/navigation/.../AppDeepLink.kt
// LINES: 36-50 — expose SAFE_ID as a public helper for route-arg validation
val SAFE_ID: Regex = Regex("^[A-Za-z0-9_-]{1,64}$")
fun safeId(raw: String?): String? = raw?.takeIf { it.matches(SAFE_ID) }

// FILE: app/.../ProofViewerScreen.kt — validate proofId, abort on bad input, encode in URL
val safeId = AppDeepLink.safeId(proofId) ?: return.also { LaunchedEffect(Unit) { onBack() } }
val url = "https://apptest.dev/v/${Uri.encode(safeId)}.png"
```

### Acceptance criteria
- [x] `AppDeepLink.safeId()` is public and used by routes
- [x] `ProofViewerScreen` rejects non-whitelist proofId and pops back
- [x] URL constructed with `Uri.encode()`

### Verification
```bash

# Expected: "fun safeId(raw: String?): String?"
grep -n "AppDeepLink.safeId" app/src/main/kotlin/com/apptest/app/proof/ProofViewerScreen.kt
# Expected: "val safeId = com.apptest.core.navigation.AppDeepLink.safeId(proofId)"
```

### Risks
| risk_id | description | mitigation |
|---|---|---|
| R-1 | other entries to ProofViewer bypass safeId (if future code calls it from elsewhere) | move the check into the route composable (Single Source of Truth) |

### Rollback
```bash

# Side-effect: ProofViewer accepts any string again; Coil request goes to whatever path is given
```

### APT-AUDIT-CRIT-004: AndroidManifest `<queries>` for Android 11+ package visibility

```yaml
id: APT-AUDIT-CRIT-004
severity: crit
status: done
owner: claude
eta_days: 0
blocker_for: [r_040_install_detection_works]
discovered_via: package visibility model on targetSdk 30+
fixed_in: pending_commit
related: [APT-AUDIT-CRIT-005, APT-AUDIT-CRIT-006]
```

### Evidence
`app/src/main/AndroidManifest.xml:12-16` — comment says "Defer to APT-V1-R-040" but R-040 is marked done in manifest.yaml. No `<queries>` block existed, so `PackageManager.getPackageInfo()` throws `NameNotFoundException` for every package on Android 11+ → `PackageInstallChecker.isInstalled()` always returns false → every active match abandoned.

### Fix
```xml
<!-- FILE: app/src/main/AndroidManifest.xml — before <application> -->
<queries>
    <intent>
        <action android:name="android.intent.action.MAIN" />
    </intent>
    <package android:name="com.android.vending" />
</queries>
```

### Acceptance criteria
- [x] `<queries>` block declared in manifest
- [ ] Manual test: install test app via Play Store closed track; AppTest's `PackageInstallChecker.isInstalled(testPkg)` returns true (requires R-040 worker to run)

### Verification
```bash
grep -A5 "<queries>" app/src/main/AndroidManifest.xml
# Expected: <intent>, <action android:name="android.intent.action.MAIN" />, </intent>, <package android:name="com.android.vending" />
```

### Risks
| risk_id | description | mitigation |
|---|---|---|
| R-1 | Play Store rejects ACTION_MAIN query as too broad | Narrow to declared test-app package list when V2 schema supports per-match package list |

### Rollback
```bash
git revert <commit_sha>
# Side-effect: install detection broken again on Android 11+
```

### APT-AUDIT-CRIT-005: Heartbeat worker drains queue while signed-out → data loss

```yaml
id: APT-AUDIT-CRIT-005
severity: crit
status: not_started
owner: tbd
eta_days: 1
blocker_for: [r_040_install_detection_correctness]
discovered_via: sub-agent audit of WorkManager / signed-out gating
fixed_in: null
related: [APT-AUDIT-CRIT-006]
```

### Evidence
- `core/data/.../worker/SupabaseHeartbeatWorker.kt:42-74` — no sign-in gate; `runCatching` swallows 401
- `core/data/.../install/UninstallEventStore.kt:35-41` — `drainAll()` clears queue BEFORE any backend call

### Fix
See parent audit deliverable (conversation) "Fix 8" for full rewrite. Summary:
1. Gate worker on `tokenProvider.token() != null` — return success early if signed-out
2. Replace `drainAll()` with `peek()` + `ack(packages)` so only confirmed-abandoned events are removed
3. Key `UninstallEventStore` by user_id (`pending_uninstalls_<uid>`)

### Acceptance criteria
- [ ] Worker returns success (not failure) when token is null
- [ ] Pending uninstalls survive signed-out worker runs
- [ ] Two users on same device cannot affect each other's queue

### Verification
```bash
grep -n "tokenProvider.token()" core/data/src/main/kotlin/com/apptest/core/data/worker/SupabaseHeartbeatWorker.kt
# Expected: sign-in gate at start of doWork()
grep -n "pending_uninstalls_\$" core/data/src/main/kotlin/com/apptest/core/data/install/UninstallEventStore.kt
# Expected: keyFor(uid) helper using user-scoped key
```

### Risks
| risk_id | description | mitigation |
|---|---|---|
| R-1 | switch from drainAll to peek+ack changes WorkManager retry semantics | document in worker KDoc; assert idempotent abandon calls server-side |

### Rollback
```bash

# Side-effect: queue data loss on signed-out worker resumes
```
